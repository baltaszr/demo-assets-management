Problem Statement

Event asset management lacks a centralized visibility layer across booth materials, POSM, inventories, and vendor storage. Teams struggle to track availability, coordinate requests, and determine which assets are recyclable vs. deployable — causing duplication, write-offs, and missed reuse opportunities.

User Stories

Brand Manager

As a brand manager, I want to see all current booths assigned to my brand so I can plan which events need new or recycled assets.

Event Operations Lead

As an ops lead, I want to request a 2×2 modular booth for an upcoming event and have the system auto-check material availability in the reuse pool.

Warehouse PIC

As a warehouse PIC, I want to be notified when an asset's development date passes the review threshold so I can move it to the reuse pool before it degrades further.

Procurement Team

As procurement, I want to view all external vendor storage slots across booth and POSM categories so I can optimize costs and reduce unnecessary vendor contracts.

Senior Management

As management, I want a top-level dashboard showing total asset value, availability rate, recycling pipeline count, and pending requests across all brands.

Acceptance Criteria — Key

AC-01 — Asset table completeness

Every asset row must carry: brand, asset type, price, warehouse type, date developed, PIC name, availability status, and open request count.

AC-02 — Reuse triggering

Assets with date-developed older than the configured threshold (default: 18 months) must appear flagged in the dashboard and auto-suggest transition to Reuse Pool.

AC-03 — Modular size coverage

Modular booths must be filterable by size (1×1, 2×2, 3×3) and linked to available reuse-pool material quantities.

AC-04 — External vendor visibility

All externally stored assets (booth and POSM) must display the vendor name in the warehouse column and be filterable separately from internal stock.
